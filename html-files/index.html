🎉 Welcome to the Git and GitHub Workshop! 🎉

Are you ready to learn how to break the internet? Just kidding, let’s start with the basics. Grab your coffee (or your therapist) because you’ll need it to survive the next few hours. 🚀

Your first task is simple: Fork this repository (don’t worry, it’s not as painful as a fork in the road), clone it, and add your amazing index page (complete with your name, university, and enough humor to make even the darkest pull request seem bearable)!

👨‍💻 Your Mission (Step-by-Step)

Follow these commands one by one, or else face the wrath of the Git Gods. 🔥✨

1. Fork this repo (click the "Fork" button at the top right of the page). No, it’s not a in this page, go to Github.
2. Clone the repo to your local machine: git clone https://github.com/your-username/your-repo-name.git
3. Create your personal index file and name it like this: your-university-index.html. No, "cool-file.html" will not do. This isn’t high school, be creative.
4. Add your fun content (name, university, something that will make us laugh – or cry, we’re flexible). 🎨
5. Add the file to the `html-files/` folder. Don’t mess with other files, it’s like walking into someone’s house and rearranging their furniture.
6. Commit your changes: git add html-files/your-university-index.html git commit -m "Add my university index page! (And I swear I didn't break anything else)"
7. Push your changes: git push origin main
8. Open a pull request to the original repo! 📬 Be sure to double-check your changes. No one likes surprises unless they’re in the form of cake.
Remember: No messing with other people's files. They’re not your toys. 🚫 Git Gods are watching!!!!

🚨 IMPORTANT RULES 🚨

Let’s make sure we don’t turn this into a disaster. Follow these rules closely, or risk being banished to the land of infinite merge conflicts (seriously, it’s a nightmare). 😱

  • 1. Don't mess with anyone else's files! Keep your hands to yourself, unless you're touching your own file. If you touch someone else's file, Git Gremlins will come after you. 👹
  • 2. Name your file like this: your-university-index.html. Don’t name it “final_v4_fixed_v2_reallyfinal.html” – that’s how you summon chaos. 🌀
  • 3. One file, one PR: If you add 10 files or change anything unrelated to your page, you'll find yourself in a deep pit of merge conflicts. You have been warned. 😈

Now go ahead, fork this project, clone your fork, and add your creative touch to the html-files/ directory! You’re not just contributing to this repo – you’re contributing to the chaos of the open-source world! 🌍🔥

⚡️ Pro Tip: Commit often, push early, and always, ALWAYS make sure your pull requests are cleaner than your room. If you're unsure, Git's got your back (and so do we). 🙌

Remember: Git is like a box of chocolates — you never know what merge conflicts you'll get. 🍫 But if you mess up, we have a secret stash of chocolate to bribe you with. 🍬
